R Package

Native R access to semisupervised mean inference with AIPW and calibration.

ppiAIPW brings the same overall workflow to R with a plain functional API, lightweight S3 result objects, calibration diagnostics, weighting utilities, and a causal wrapper for arm-specific means and Wald ATEs.

R Quickstart Open Package Source

Main Entry Point

Function mean_inference(...)
Default Method method="monotone_spline"

Functional, console-friendly, and aimed at biostats, epi, and applied ML workflows in R.

Minimal Example

library(ppiAIPW)

result = mean_inference(
    Y,
    Yhat,
    Yhat_unlabeled,
    method = "monotone_spline",
    alpha = 0.1
)

print(result)
summary(result)

Install

Install the R package from GitHub or from a local checkout

The R package lives in the repository subdirectory r/ppiAIPW, so GitHub installs should target that subdirectory explicitly.

GitHub install with remotes

remotes::install_github(
  "Larsvanderlaan/ppi-aipw",
  subdir = "r/ppiAIPW"
)

GitHub install with pak

pak::pak("Larsvanderlaan/ppi-aipw/r/ppiAIPW")

Local source install

R CMD INSTALL r/ppiAIPW

Quickstart

Start with mean_inference(...)

The R package follows the same conceptual API as the Python package, but with R-native functional usage and S3 result objects. Use mean_inference(...) when you want the point estimate, standard error, confidence interval, fitted calibrator, and diagnostics together. 

library(ppiAIPW)

result = mean_inference(
    Y,
    Yhat,
    Yhat_unlabeled,
    method = "monotone_spline",
    alpha = 0.1
)

estimate = result$pointestimate
standard_error = result$se
lower = result$ci[[1]]
upper = result$ci[[2]]

summary(result)

mean_pointestimate()

Return only the point estimate when you do not need the full result object.

mean_se() and mean_ci()

Pull out uncertainty summaries directly for scripting or reporting pipelines.

method="auto"

Choose among "aipw", "linear", "monotone_spline", and "isotonic" by cross-validated influence-function variance.

calibration_diagnostics()

Inspect binned labeled-sample calibration and plot the fitted curve from a result or fitted model.

causal_inference()

Estimate arm-specific means and control-vs-treatment ATEs from predicted potential outcomes in one R call.

compute_two_sample_balancing_weights()

Construct nonnegative labeled-sample balancing weights for two-sample mean problems.

Parity

Close to the Python package, but designed for R users

The target is close functional parity and the same practical workflow, not byte-for-byte identical optimizer traces.

What is shared

The same mean-inference problem setup, the same major method families, the same default recommendation, and the same causal convenience layer.

What feels different in R

Functions return list-like S3 objects with print(), summary(), and plot() methods instead of Python result classes.

When to use this page

If your team works in RMarkdown, Quarto, or standard R analysis scripts and wants a native package rather than a Python bridge.
