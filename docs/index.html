Semisupervised Mean Inference For Applied ML

Semisupervised mean inference with AIPW, calibration, and practical uncertainty quantification.

ppi_aipw is a practitioner-friendly package for semisupervised mean estimation with labeled outcomes, a larger unlabeled pool, and predictions from a fixed model. It combines AIPW, post-hoc calibration, and practical uncertainty quantification within the broader semiparametric and debiased machine learning toolkit.

Recommended Default

Estimator method="monotone_spline"
Uncertainty inference="wald"

A strong default for many workflows: smooth, monotone, and less stepwise than isotonic calibration.

Recommended API

from ppi_aipw import mean_inference

result = mean_inference(
    Y, Yhat, Yhat_unlabeled,
    method="monotone_spline",
    alpha=0.1,
    inference="wald",
)

estimate = result.pointestimate
standard_error = result.se
lower, upper = result.ci

The narrower wrappers mean_pointestimate, mean_se, and mean_ci are still available when you only want one output at a time.

Quickstart

What the arguments mean

The main API is small. You provide labeled outcomes, labeled predictions, unlabeled predictions, and your choices for calibration and interval construction. The recommended entrypoint is mean_inference(...), which returns the point estimate, standard error, confidence interval, calibrator, and method-selection diagnostics together.

If you want a data-adaptive choice among a few methods, set method="auto" and pass candidate_methods=("aipw", "linear", "isotonic"). By default it uses num_folds=100 and min(n_unlabeled, 10 * n_labeled) unlabeled rows in the foldwise objective, while the final point estimate still uses the full unlabeled sample.

Y

Observed outcomes for the labeled sample.

Yhat

Predictions on the same labeled rows.

Yhat_unlabeled

Predictions on the unlabeled sample.

method

Choose "aipw", "linear", "prognostic_linear", "sigmoid", "monotone_spline", "isotonic", or "auto".

candidate_methods

Candidate methods considered when method="auto" minimizes cross-validated influence-function variance. If "aipw" is included, the selector also compares an efficiency-maximized AIPW candidate.

num_folds

Number of folds used by method="auto". The default is 100 and it is capped at the labeled sample size.

auto_unlabeled_subsample_size

Unlabeled subset size used by method="auto" during foldwise selection and cross-fitted lambda estimation. The default is min(n_unlabeled, 10 * n_labeled).

inference

Choose "wald" for speed or "bootstrap" for a resampling-based interval that refits calibration inside each replicate.

efficiency_maximization

Optional rescaling to lambda m(X), where m(X) is the raw score for method="aipw" and the calibrated score otherwise. The package chooses lambda by empirical efficiency maximization.

isocal_backend

Backend for method="isotonic". The default is a one-round monotone XGBoost fit with isocal_min_child_weight=10; use "sklearn" for scikit-learn isotonic regression.

w, w_unlabeled

Optional observation weights for labeled and unlabeled samples.

X, X_unlabeled

Optional extra covariates used by method="prognostic_linear". The score and intercept stay unpenalized, while the extra covariates get ridge tuning on the labeled sample.

      Historical Review

      Historical context for PPI within semisupervised inference

      This section is based on the historical framing in your paper. The goal is to place recent prediction-powered inference methods within the broader semiparametric, missing-data, debiased machine learning, and flexible covariate-adjustment literatures, while highlighting how calibration connects these perspectives in a useful way.

      1970s-2000s

      Missing data and semiparametric efficiency came first

      Long before “prediction-powered inference,” semiparametric statistics developed influence-function methods, doubly robust estimators, and AIPW-style procedures for missing-at-random outcomes. In your paper’s framing, the semisupervised mean problem is one instance of that older missing-outcome setup.

      1990s-2020s

      Debiased ML made black-box prediction compatible with valid inference

      Debiased machine learning, TMLE, and related orthogonal methods showed how to combine flexible prediction models with valid inference. In the semisupervised mean setting, these ideas provide a natural interpretive framework for AIPW-style estimators built from black-box prediction models.

      Recent ML framing

      PPI popularized the setup for modern black-box predictors

      Recent ML papers recast the same broad problem as prediction-powered inference: a small labeled sample, a large unlabeled sample, and a useful but imperfect predictor. Your paper emphasizes that this modern framing is valuable for applications, while also being closely connected to established AIPW and efficiency-based perspectives.

      This paper’s synthesis

      Calibration helps position PPI within familiar semiparametric methods

      Your paper’s central synthesis is that recent prediction-powered inference methods belong squarely to the standard semiparametric and debiased machine learning toolkit for randomized trials and missing-at-random settings. In particular, PPI++ can be viewed as AIPW with empirical efficiency maximization and, in the one-score setting, also as linear calibration and classical prognostic-score regression adjustment.

      Positioning within existing semiparametric methods

      The paper’s positioning argument is not that prediction-powered inference is unimportant, but that it is most informative when read as a concrete instance of familiar semiparametric ideas. In this view, semisupervised mean estimation connects directly to older missing-data, randomized-trial, and debiased machine learning methods, including work that learns prognostic scores from larger auxiliary data sets and then uses them for regression adjustment in smaller target samples.

      This perspective also suggests a practical research direction: semisupervised inference may benefit from closer translation of tools developed for missing-at-random problems, randomized trials, and debiased machine learning, rather than being treated as a completely separate inferential paradigm.

      Empirical Takeaways

      What the paper and package checks suggest in practice

      Default workflow

      Start with monotone spline calibration and Wald intervals.

      Binary or bounded scores

      Sigmoid calibration is a sensible robustness check when a smooth sigmoid recalibration is plausible.

      Nonlinear miscalibration

      Isotonic calibration is attractive when monotonicity is believable and labeled data are large enough to fit a stable curve.

      Where PPI fits

      PPI and PPI++ fit naturally within the broader semiparametric and debiased machine learning toolkit for semisupervised mean estimation.

      References

      Where this website’s framing comes from

      The historical review and practitioner recommendations on this page are drawn from the paper text in paper/main.tex in this repository.

      Main paper themes reflected here:

      • PPI and PPI++ can be understood within the AIPW, semiparametric-efficiency, and debiased machine learning framework.
      • In the one-score setting, PPI++ aligns algebraically with linear calibration and prognostic-score regression adjustment.
      • Calibration is stronger than mean-bias correction alone and can improve semisupervised inference when the score is miscalibrated.
      • Monotone spline calibration is the package's practical default, while linear remains the simplest affine alternative and isotonic calibration is a useful more flexible monotone upgrade when labeled data support it.
      • Bootstrap intervals are a practical complement to Wald intervals.
